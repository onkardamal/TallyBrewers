# kiro Rules

Always follow Clean Architecture.

Never generate fake code.

Never use mock data.

Never create unnecessary features.

Never rewrite working code.

Always explain every implementation.

Always update:

TASK_LOG.md

CURRENT_STATE.md

SYSTEM_ARCHITECTURE.md

after every completed task.

Keep code modular.

Keep code production-ready.

Always ask before changing architecture.

Prioritize simplicity over features.

Build exactly what PROJECT_BRIEF.md describes.

Never introduce:
- Risk Engine
- Authorization Workflow
- Admin Dashboard
- Team Management
unless explicitly requested.
